## 1. 产品概述

### 1.1 一句话定义

天玑是天枢系统的**全球信息采集底座**，完全自动化，只做采集 + 结构化入库 + PDF报告输出，**不可被人类污染**。

### 1.2 定位

```
天枢系统三层架构：

天玑（底座）  →  开阳（执行层）  →  玉衡（决策层）
  采集            分析+执行          人类判断
  原始            自动化             不可替代
  不可污染         可判断              定方向
```

### 1.3 核心原则

| 原则 | 说明 |
|------|------|
| 完全自动化 | 无需人类触发，定时运行 |
| 不可污染 | 原始数据只存不改，分析在天玑之外 |
| 三层API | 官方API→免费开源→付费兜底 |
| 全球覆盖 | 不做地域裁剪，121个平台起步 |
| 结构化入库 | 每条记录带checksum/版本/时间戳 |
| 分级告警 | 🔴开阳处理 / 🟡关注 / 🟢归档 |

### 1.4 用户画像

| 用户 | 角色 | 使用方式 |
|------|------|---------|
| 开阳 | 自动化执行层 | 接收天玑日报→自动分析→响应/升级 |
| JUN (玉衡) | 人类决策者 | 接收开阳上报的"待判断"节点→做决策 |

完整平台清单见 `references/tianji-platform-map.md`

### 2.2 采集分层策略

```
第一层：官方自有API（首选）
  ├─ 数据可信，有合同约束
  ├─ 但有限速/付费/审核门槛
  └─ 例：Reddit API / X API / Amazon SP-API / FRED API

第二层：免费开源（无封号风险）
  ├─ RSS feeds、公开数据集
  ├─ 不会被封IP或账号
  └─ 例：Google Alerts RSS / Government RSS / Wikipedia API

第三层：付费API集合（兜底）
  ├─ 聚合服务，一条API多个源
  ├─ 但有成本，且数据经过二次加工
  └─ 例：RedFox（国内社媒）、Freightos、Trading Economics

国内桥接：RedFox API（已付费）
  ├─ 小红书、抖音、微博、知乎等
  └─ 快速验证，绕过审核流程
```

### 2.3 降级策略

```
API正常 → 用官方数据（最佳质量）
API限流 → 降级到免费替代（RSS/缓存）
API失效 → 用付费兜底（RedFox/聚合）
全挂    → 标记采集失败 + 日志告警
```

## 3. 数据存储架构

### 3.1 五盘位分配

| 盘位 | 用途 | 数据内容 | 访问模式 | 保留期限 |
|------|------|---------|---------|---------|
| **盘1** | 原始数据 | 天玑采集的原始JSON/XML/HTML，只写不删 | 写入后只读 | 永久 |
| **盘2** | 结构化数据库 | 归一化后的结构化记录，带索引 | 读/追加 | 永久 |
| **盘3** | PDF报告归档 | 日报/周报/月报PDF | 读/写 | 2年 |
| **盘4** | 系统备份 | 系统配置、代码、数据库快照 | 定时备份 | 按版本 |
| **盘5** | 冷数据/日志 | 历史归档、采集日志、调试信息 | 低频访问 | 永久(冷) / 90天(日志) |

### 3.2 数据库schema（初稿）

```sql
-- 原始数据表（盘1）
CREATE TABLE tianji_raw (
    id BIGSERIAL PRIMARY KEY,
    domain VARCHAR(20) NOT NULL,       -- regulatory / social / marketplace / ...
    source VARCHAR(50) NOT NULL,        -- nsf / reddit / amazon / ...
    source_type VARCHAR(20) NOT NULL,   -- api / rss / scraper / redfox / paid_api
    url TEXT,
    raw_data JSONB NOT NULL,            -- 原始数据（不可修改）
    checksum VARCHAR(64) NOT NULL,      -- SHA256
    fetched_at TIMESTAMPTZ NOT NULL,
    http_status INT,
    latency_ms INT,
    version INT DEFAULT 1,
    UNIQUE(checksum)                    -- 防止重复写入
);

-- 结构化数据表（盘2）
CREATE TABLE tianji_structured (
    id BIGSERIAL PRIMARY KEY,
    raw_id BIGINT REFERENCES tianji_raw(id),
    domain VARCHAR(20) NOT NULL,
    source VARCHAR(50) NOT NULL,
    priority_score INT DEFAULT 0,
    signal_type VARCHAR(30),            -- regulation_change / price_alert / trend_shift / ...
    signal_content TEXT,
    impact_level VARCHAR(10),           -- critical / high / medium / low / info
    status VARCHAR(20) DEFAULT 'new',   -- new / confirmed / escalated / archived
    ai_summary TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    resolved_at TIMESTAMPTZ
);

-- 报告表（盘3）
CREATE TABLE tianji_report (
    id BIGSERIAL PRIMARY KEY,
    report_type VARCHAR(10),            -- daily / weekly / monthly
    report_date DATE NOT NULL,
    pdf_path TEXT,
    alert_count INT DEFAULT 0,
    signal_count INT DEFAULT 0,
    coverage_pct DECIMAL(5,2),
    generated_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 8. 边界与红线

### 红线（不可触碰）

| 红线 | 违反后果 | 保护措施 |
|------|---------|---------|
| 原始数据不可修改 | 数据污染→信息决策错误 | 盘1只读挂载+checksum校验 |
| 天玑不做任何分析 | 超出范围→与开阳责任混淆 | 所有规则在开阳Skill定义 |
| 人类不可直接修改天玑记录 | 人类污染→信号失真 | 有异议走"标注"流程 |
| 采集失败必须有降级 | 单点故障→数据中断 | 三层API兜底 |
| 每天下班前出报告 | 错过当天决策窗口 | 调度器冗余+16:30强制触发 |

### 边界

| 天玑做 | 天玑不做 |
|--------|---------|
| 定时采集原始数据 | 分析数据含义（开阳做） |
| 结构化入库 | 做决策（玉衡做） |
| 生成PDF报告 | 执行操作（开阳做） |
| 标注入库时间/来源/checksum | 标注"倾向性判断" |
| 检测到异常触发深度采集 | 决定告警是否升级为行动（开阳做） |
| 记录采集失败日志 | 自动修复故障（人工介入） |
